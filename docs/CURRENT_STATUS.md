# 当前状态

更新日期：2026-04-03

## 当前项目状态

项目已经是一个可直接打开使用的静态前端原型，核心 UI、视觉风格切换、状态持久化与基础交互都已经成型；同时已补上 Vercel Serverless 搜索联想接口，用于承接更接近浏览器 omnibox 的词根和联网联想。

## 已经稳定的部分

- 根目录静态文件可直接运行：`index.html`、`styles.css`、`script.js`
- 已接入桌面端搜索联想弹层，联想窗口沿用液态玻璃视觉
- 搜索联想已支持“本地记录 + 词根扩展 + 联网建议 + 内容结果”的混合排序
- Vercel 生产环境已可稳定返回真实联想结果，GitHub 与部署链路已打通
- 状态读写已有统一框架，避免新线程继续把持久化逻辑写散
- 已接入现成液态玻璃相关库，减少重复造轮子
- 现有 README 已经能说明项目用途与基本使用方式

## 当前结构上的真实情况

- 仓库里有一个 `app/` 目录，但当前可用实现仍然以根目录静态文件为主
- 新增 `api/` 目录承载 Vercel Serverless Functions，目前主要提供 `/api/suggest`
- 线程之间的上下文之前主要依赖人工口头说明，缺少统一交接入口
- 本次已经开始补齐跨线程文档系统

## 当前最重要的约束

- 任何新增持久化字段都要先走状态框架
- 搜索联想的前端降级路径要保留，远端接口失败时不能影响本地联想与基本搜索
- 功能变更完成后，应同步更新文档中的“当前状态”和“下一步”
- 如果一次线程涉及较复杂分析、试验或交接，应留下单独线程记录

## 本次建立文档系统后的预期状态

- 新线程可以只看文档就知道项目是什么、做到哪了、接下来做什么
- 重要决策和进展能被沉淀到仓库，而不是停留在单一对话里
- 后续再开线程时，可以优先读文件而不是重新问项目背景
